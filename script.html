    Description

    We can add a article in the existing replication by the following given queries. These queries are executed step by step to avoid any issue. This query has 3 specific steps mentioned below.
    1. Changes Publication properties – In the step the two properties value of the publication are changed to false. So that the user can add the article in the existing replication. The properties are “allow_anonymous” and “'immediate_sync”.
    2. Add Article—This step adds the article into the replication. We just need to provide the parameters like article name, Publication Name, Pre-Creation commands, schema option etc.
    3. Refresh The Publication – This command refreshes the publication by executing a SP with the publication name.

    There can be 4th step too in case of snapshot replication. We need to run the snapshot agent after performing the above 3 steps, but if replication type is support only, then we do not require to run the snapshot agent.

    Description

    1. Check Pending Commands – Query is used to check pending commands from distributor to subscriber. The values to the parameters are provided specific to the publication, subscription, publisher db , subscription db and replication. These parameters will differ accordingly.
    2. Dist To Sub Latency – The given query is used to provide the Latency of commands from distributor to subscriber end. 3. Pub To Sub Latency – We use the proc to check the latency from publisher db to subscriber db.
    4. Replicating Columns – The query is used when the user needs to check the replication columns of a specific table in a publication.
    5. Script Out Custom Procs – Sometimes there can be an error of some missing proc on the subscription end. Then by using the mention proc with the publication name we can get the script of that proc by running the query on publisher DB.
    6. When Subscription is Inactive – If user needs to know the subscription state, then it can be found by the given select query and if it is inactive means 0 then can be made active by given update query.
    7. Table Which Are Part of Replication – The query result set provides the list of the tables which are part of replication in the selected db.
    8. To Get Transaction Count – We can use the given queries to get the count of the transaction from distribution db.
